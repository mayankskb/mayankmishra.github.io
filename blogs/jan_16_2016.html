Data Structures : What it is ?

Mayank Mishra

In this era of Computer Science, we all running behind the technology such as Java, PHP, Python, ruby and many more and due to this running, we forget the plenary or requisite method of doing programming. With the advancement of the syllabus, a metric student can also do the programming either it is related to C, C++, Java or web site designing or developing.

From my point of view, this is not the right way of doing code. Then what is the right way of doing code or programming? Isn’t this is the question that we should think… Well don’t take the strain, I am here to share my point of view regarding the lawful (in terms of computer science) way of doing code.

See each language of each domain deals with the data right, So at first, we must understand that what data is. As per most of us, Data is the collection of raw facts and figures. Absolutely right but I would like to add few more words "used for analysis, calculation, and for research work in order to give a result".

Now what a programmer requires to do with data, obviously deals. A programmer deals with data for its input, output, processing and storing of data. See here are two terms processing and storing of data for these two we need to map the data using a proper fashion (in this era of fashion data also wants to do fashion). This is what we call as mapping of data and is termed as Data Structures.

Not only data structures we also need algorithms for programming. Summing this whole as

Data Structures + Algorithm = Programming


We can map data using various Data Structures such as Arrays, Linked Lists, Trees, Stacks, Queues, Graph, Forests, etc.

For the beginning, we must know some common approaches of algorithms such as Sorting, Searching, Hashing, String Matching and exponentiation, Dynamic programming and at least one computer programming language to implement these tactics and Data Structures.
